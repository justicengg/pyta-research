# 研究画布交互设计说明（初稿）

## 1. 文档目标

本文档用于沉淀当前阶段已经确认的 UI/UX 方向，作为后续前端实现、交互微调和多 Agent 沙盘推演继续演进的基础说明。

## 2. 核心设计哲学

### 2.1 产品不是传统 SaaS 后台

目标不是做一个需要频繁跳页、切 tab、点表单的传统 SaaS 平台，而是做一个围绕研究、推演、决策持续展开的统一工作界面。

### 2.2 一张持续存在的研究画布

整个系统的核心不是页面流，而是一张持续存在的研究画布。

用户在同一界面内完成：
- 输入信息
- 接入来源
- 沙盘推演
- 多 Agent 协作
- 输出决策
- 调整模式和工具

### 2.3 简洁优先，功能隐藏到第二层

页面第一层尽量少按钮、少显性控制项。
复杂能力应尽量收进：
- 左右边栏
- 对话框内部
- `+` 菜单
- 设置抽屉或安全区

### 2.4 开放但安全

未来会支持用户自定义 Agent、外部 Agent、更多数据源和技能。
但敏感配置不能直接裸露给模型层。

因此：
- 普通控制可走对话框
- 敏感字段进入左侧安全设置区

---

## 3. 整体布局

整个界面采用三段式结构：

1. 左侧边栏：信息层
2. 中间主区域：研究画布 / 沙盘推演舞台
3. 右侧边栏：决策架

中间区域底部附着统一对话框（Command Console）。

---

## 4. 左侧边栏：信息层

### 4.1 职责

左侧边栏负责输入上下文，不承担最终推演输出。

主要内容包括：
- 已接入来源
- 实时事件流
- 参考资料
- 搜索
- 安全配置

### 4.2 安全配置

敏感配置应留在左侧边栏的安全区域，不通过底部对话框直接传递。

典型敏感信息包括：
- API Key
- Token
- 私有 Endpoint
- 企业内部接入信息

设计约束：
- 安全能力与普通自然语言控制分离
- 模型可指导配置，但不默认读取敏感明文

### 4.3 收起行为

左侧边栏支持点击收起。
收起后不保留完整面板，只保留一个小点。
点击小点可以重新展开。

### 4.4 布局联动

左侧边栏收起时：
- 中间画布宽度自动扩大
- 底部对话框随画布自适应扩大
- 视觉保持对称和简洁

---

## 5. 右侧边栏：决策架

### 5.1 职责

右侧边栏不承载完整推演过程，只承载推演收敛后的结果。

主要内容包括：
- 当前摘要
- 风险提示
- 动作队列
- 决策卡
- 执行建议

### 5.2 为什么不把 Agent 放在右栏

多 Agent 的推演过程应该留在中间画布中可视化呈现。
右侧只负责承接结果，不负责承接过程。

也就是说：
- 过程在中间
- 收敛结果在右边

### 5.3 收起行为

右侧边栏也支持点击收起。
收起后同样变成一个小点，可再次展开。

### 5.4 布局联动

右侧边栏收起时：
- 中间画布自动变宽
- 底部对话框跟随中间画布自适应

---

## 6. 中间主区域：研究画布 / 沙盘推演舞台

### 6.1 定位

中间区域不是普通内容容器，而是整个系统的核心工作记忆区。

它同时承担：
- 核心研究对象展示
- 多 Agent 推演
- 场景编排
- 观点生成
- 中间判断沉淀
- 决策草稿形成

### 6.2 视觉方向

最终采用：
- 舞台化骨架
- 时间流隐线

而不是：
- 传统流程图
- 纯表格式泳道
- 厚重后台卡片墙

### 6.3 中心舞台

画布中心保留当前推演对象和核心场景。

例如：
- 腾讯 / 港股科技
- 当前推演周期
- 当前主场景

设计目标：
- 保持呼吸感
- 让用户一眼知道当前关注对象
- 让所有 Agent 围绕核心对象组织推演

### 6.4 多 Agent 分布

多个 Agent 围绕中心对象分布，包括但不限于：
- 沙盘 Agent
- 宏观 Agent
- 情绪 Agent
- 用户 Agent
- 决策 Agent

这些 Agent 不应只是边栏中的一串名字，而应作为画布中的参与者存在。

### 6.5 过程表达

推演过程不做硬泳道，而通过轻痕迹表达：
- 节点之间细线连接
- 最近动作高亮
- 历史步骤逐渐淡化
- 当前活跃步骤最清晰

目标是：
- 用户能看懂过程
- 但不会满屏变成流程框

### 6.6 节点类型

当前初稿中已验证的节点包括：
- 核心场景节点
- Agent 节点
- 事件节点
- 中间判断节点
- 宏观修正节点
- 当前活跃步骤节点
- 决策草稿节点
- 图例节点

这些对象属于“思考过程对象”，不是传统页面卡片。

---

## 7. 画布交互

### 7.1 节点自由拖拽

中间画布中的核心对象支持自由拖拽。
当前包括：
- 中心场景
- 各个 Agent
- 各类事件/判断节点
- 决策草稿
- 图例

目的：
- 提升沙盘感
- 强化工作记忆组织能力
- 让用户按自己的思维结构摆放对象

### 7.2 工具条可拖拽

中间画布中的工具条也应支持自由拖拽。
例如：
- 添加事件
- 添加 Agent
- 添加场景
- 添加观点
- 连接来源
- 运行推演

### 7.3 画布缩放与平移

画布支持基础空间操作：
- 滚轮/触控板缩放
- 拖动画布空白区域平移

这样用户可以：
- 放大查看局部推演结构
- 缩小看整体布局
- 在多 Agent 场景中左右移动查看

当前优先采用自然交互，不强加一个厚重的坐标控制箭头。
如后续测试需要，再补轻量方向控制器。

---

## 8. 底部对话框（Command Console）

### 8.1 定位

底部区域不是普通聊天框，而是统一控制台。

它负责：
- 输入指令
- 调整推演过程
- 调用工具
- 添加能力
- 切换模式
- 驱动画布操作

### 8.2 位置要求

对话框位于中间主画布的底部区域。
其宽度和位置应始终与中间画布对称、自适应联动。

不能出现：
- 左右边栏变了，但对话框仍固定在旧位置
- 画布变宽了，对话框不跟着调整
- 明显不对称的视觉失衡

### 8.3 结构要求

对话框内部结构为：
- 上方：文本输入区域
- 下方左侧：`+` 与模式选择

不使用硬分隔线把文本区和工具区粗暴切开。

### 8.4 `+` 菜单

`+` 菜单用于收纳第二层功能，避免第一层按钮过多。

包含但不限于：
- 连接来源
- 加载技能
- 加载 Agent
- 场景设置
- 安全配置入口

### 8.5 模式切换

模式切换要尽量轻，不做厚重 button。
当前确认模式：
- 极简模式（默认）
- 专家模式

样式要求：
- 更像文本选择器
- 用轻量下拉箭头
- 默认优先极简模式

### 8.6 安全职责边界

底部对话框不再承担敏感字段直接输入职责。
敏感配置回到左侧安全设置。

---

## 9. 已确认的关键交互约束

### 9.1 少按钮原则

- 尽量减少第一层按钮
- 把复杂能力隐藏进第二层
- 保持页面清洁与呼吸感

### 9.2 边栏与画布联动

- 边栏收起后变小点
- 中间画布自动扩展
- 底部对话框自动扩展并保持对称

### 9.3 画布优先

- 中间画布始终是视觉和功能核心
- 不再采用多个大卡片割裂布局
- 研究对象、Agent、推演过程都统一放进同一块画布空间

### 9.4 多 Agent 可扩展性

当前原型只验证结构，不代表最终视觉定稿。
后续增加多个 Agent 后，可能还需要继续弱化“卡片感”，进一步向“轻节点 + 舞台化 + 时间流隐线”的最终方向演进。

---

## 10. 当前阶段不做的内容

本轮原型先不做：
- 完整业务后端联动
- 真实来源接入流程
- 真实 Agent 调度逻辑
- 企业级权限系统
- 最终态多 Agent 视觉精修
- 复杂动画系统

---

## 11. 当前结论

这版可以作为当前阶段的交互界面初稿。

当前初稿已经具备：
- 左右边栏
- 中间融合式研究画布
- 底部统一控制台
- 多 Agent 舞台化 + 时间流轻痕迹
- 节点拖拽
- 工具条拖拽
- 画布缩放和平移
- 边栏收起小点
- 画布与对话框自适应联动

后续工作重点不再是大改架构，而是：
- 视觉细化
- 多 Agent 场景下的密度控制
- 真实推演测试后的交互微调
